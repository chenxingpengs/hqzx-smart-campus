## 1. 系统概述

### 1.1 系统简介

智慧校园考勤系统是一套完整的校园晚午托考勤管理解决方案，涵盖微信小程序、桌面客户端、Web管理后台三个前端平台，以及统一的Flask后端服务。系统支持教师考勤打卡、学生签到签退、排班管理、通知推送、设备集控等功能。

### 1.2 系统功能

| 功能模块 | 描述                 |
| ---- | ------------------ |
| 用户认证 | 微信登录、账号绑定、Token管理  |
| 考勤管理 | 考勤打卡、考勤查询、考勤统计     |
| 排班管理 | 周期排班规则、特殊值班日历、时段配置 |
| 设备绑定 | 桌面客户端扫码绑定          |
| 通知推送 | 实时通知、寻人传呼、广播消息     |
| 设备集控 | 远程开关机、批量控制、状态监控    |
| 插件系统 | 可扩展插件架构            |

### 1.3 用户角色

| 角色           | 权限描述             |
| ------------ | ---------------- |
| 管理员 (admin)  | 系统全部功能、用户管理、权限配置 |
| 教师 (teacher) | 考勤打卡、班级管理、通知发送   |
| 学生 (student) | 查看个人考勤记录         |

#### 3.1.2 配置说明

配置文件位于 `config.py`，通过环境变量加载：

```python
class Config:
    SECRET_KEY = os.environ.get('SECRET_KEY')      # JWT密钥
    WX_APPID = os.environ.get('WX_APPID')          # 微信小程序AppID
    WX_SECRET = os.environ.get('WX_SECRET')        # 微信小程序Secret
    
    DB_HOST = os.environ.get('DB_HOST')            # 数据库主机
    DB_USER = os.environ.get('DB_USER')            # 数据库用户
    DB_PASSWORD = os.environ.get('DB_PASSWORD')    # 数据库密码
    DB_NAME = os.environ.get('DB_NAME')            # 数据库名称
    
    REDIS_HOST = os.environ.get('REDIS_HOST')      # Redis主机
    REDIS_PORT = int(os.environ.get('REDIS_PORT', 6379))
    REDIS_PASSWORD = os.environ.get('REDIS_PASSWORD')
```

#### 3.1.3 数据库模型

**用户模型 (User)**

| 字段             | 类型           | 说明                        |
| -------------- | ------------ | ------------------------- |
| id             | Integer      | 主键                        |
| user\_code     | String(30)   | 学工号                       |
| username       | String(50)   | 姓名                        |
| password       | String(100)  | 密码(加密)                    |
| role           | Enum         | 角色(student/teacher/admin) |
| class\_id      | Integer      | 班级ID                      |
| wechat\_openid | String(100)  | 微信OpenID                  |
| permission     | SmallInteger | 权限级别                      |

**考勤记录模型 (AttendanceRecord)**

| 字段               | 类型      | 说明   |
| ---------------- | ------- | ---- |
| id               | Integer | 主键   |
| schedule\_id     | Integer | 排班ID |
| class\_id        | Integer | 班级ID |
| teacher\_id      | Integer | 教师ID |
| attendance\_date | Date    | 考勤日期 |
| should\_attend   | Integer | 应到人数 |
| actual\_attend   | Integer | 实到人数 |
| leave\_count     | Integer | 请假人数 |
| absent\_count    | Integer | 缺勤人数 |

#### 3.1.4 API路由前缀

| 前缀           | 用途          |
| ------------ | ----------- |
| `/app/*`     | 小程序接口       |
| `/desktop/*` | 桌面客户端接口     |
| `/web/*`     | 管理后台接口      |
| `/socket.io` | WebSocket连接 |

#### 3.1.5 认证机制

系统采用 JWT (JSON Web Token) 进行身份认证：

1. **Access Token**: 有效期1天，用于API请求认证
2. **Refresh Token**: 有效期30天，用于刷新Access Token

请求头格式：

```
Authorization: Bearer <access_token>
```

## 4. 用户使用手册

### 4.1 小程序端使用指南

#### 4.1.1 登录与绑定

**首次使用：**

1. 打开微信，扫描小程序码或搜索小程序名称
2. 点击「微信登录」按钮
3. 授权微信登录后，进入账号绑定页面
4. 输入学工号和密码，点击「绑定」
5. 绑定成功后自动跳转首页

**后续登录：**

- 已绑定账号的用户，打开小程序即可自动登录

#### 4.1.2 首页功能

首页展示以下内容：

| 区域    | 功能                  |
| ----- | ------------------- |
| 顶部信息栏 | 显示日期、用户信息、扫码按钮      |
| 快捷入口  | 考勤管理、发送通知、寻人传呼、应用中心 |
| 通知公告  | 系统公告和通知             |
| 常用功能  | 功能快捷卡片              |

**扫码功能：**

- 点击右上角扫码图标
- 扫描桌面客户端二维码
- 确认绑定设备

#### 4.1.3 考勤管理

**我的考勤：**

1. 进入「考勤管理」页面
2. 默认显示「我的考勤」标签
3. 查看今日排班和考勤状态
4. 点击「打卡」按钮完成考勤

**考勤查询：**

1. 切换到「考勤查询」标签
2. 选择日期范围、班级
3. 点击「查询」查看考勤记录
4. 支持导出考勤数据

**排班管理：**

1. 切换到「排班」标签
2. 查看个人排班日历
3. 申请换班：
   - 点击要换班的日期
   - 选择可换班人员
   - 填写换班原因
   - 提交申请

**换班处理：**

- 收到换班申请时会弹出提示
- 查看申请详情
- 选择「同意」或「拒绝」

#### 4.1.4 个人中心

1. 点击底部「我的」标签
2. 查看和修改个人信息：
   - 头像设置
   - 昵称修改
   - 手机号绑定
   - 密码修改

#### 4.1.5 考勤详情填写

**进入方式：**

- 从首页快捷入口进入
- 从考勤管理页面点击考勤记录进入

**功能说明：**

1. **考勤详情页面** (`/pages-attendance-detail/index`)
   - 显示当日考勤时段信息
   - 填写应到人数、实到人数
   - 记录请假、缺勤学生名单
   - 提交考勤记录
2. **我的班级** (`/pages-attendance-detail/MyClass`)
   - 查看班级学生列表
   - 查看学生详细信息
   - 快速筛选学生
3. **班级考勤** (`/pages-attendance-detail/MyClassAttendance`)
   - 批量签到/签退
   - 按学生逐一考勤
   - 查看考勤统计

**操作步骤：**

1. 选择考勤日期和时段

### 6.5 Nginx配置

```nginx
server {
    listen 443 ssl;
    server_name hqzx.me;

    # SSL配置
    ssl_certificate /path/to/cert.pem;
    ssl_certificate_key /path/to/key.pem;

    # 管理后台
    location / {
        root /var/www/admin;
        try_files $uri $uri/ /index.html;
    }

    # API代理
    location /app/ {
        proxy_pass http://127.0.0.1:5000;
        proxy_set_header Host $host;
        proxy_set_header X-Real-IP $remote_addr;
    }

    location /web/ {
        proxy_pass http://127.0.0.1:5000;
        proxy_set_header Host $host;
        proxy_set_header X-Real-IP $remote_addr;
    }

    location /desktop/ {
        proxy_pass http://127.0.0.1:5000;
        proxy_set_header Host $host;
        proxy_set_header X-Real-IP $remote_addr;
    }

    # WebSocket代理
    location /socket.io/ {
        proxy_pass http://127.0.0.1:5000;
        proxy_http_version 1.1;
        proxy_set_header Upgrade $http_upgrade;
        proxy_set_header Connection "upgrade";
        proxy_set_header Host $host;
    }
}
```

## 7. 常见问题

### 7.1 小程序问题

**Q: 登录后提示"未绑定账号"？**

A: 首次使用需要绑定系统账号。点击「绑定账号」，输入学工号和密码完成绑定。

**Q: 扫码绑定设备失败？**

A: 请确保：

1. 设备已启动并显示二维码
2. 二维码未过期
3. 网络连接正常

**Q: 考勤打卡失败？**

A: 检查以下项：

1. 当前时间是否在考勤时段内
2. 是否有排班安排
3. 网络连接是否正常

### 7.2 桌面客户端问题

**Q: 程序无法启动？**

A: 检查：

1. 是否已安装 .NET 8 Runtime
2. 杀毒软件是否拦截
3. 查看日志文件排查错误

**Q: 无法接收通知？**

A: 检查：

1. WebSocket连接状态
2. 网络是否正常
3. 是否被防火墙拦截

**Q: 小部件不显示？**

A:

1. 检查小部件设置是否启用
2. 重启程序
3. 检查配置文件

### 7.3 管理后台问题

**Q: 登录失败？**

A: 检查：

1. 用户名密码是否正确
2. 账号是否被禁用
3. 清除浏览器缓存重试

**Q: 设备显示离线但实际在线？**

A:

1. 检查客户端WebSocket连接
2. 刷新页面
3. 检查后端服务状态

**Q: 批量操作部分失败？**

A: 查看操作结果提示，可能原因：

1. 部分设备离线
2. 部分设备无MAC地址(开机操作)
3. 网络超时
